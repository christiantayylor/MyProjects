# Vacation Planner — Maintenance User Guide
**For Developers and System Administrators**

---

## 1. Prerequisites

The following software must be installed before setting up the application:

| Software | Minimum Version | Download |
|---|---|---|
| Node.js | v18.0.0 or higher | https://nodejs.org |
| npm | v9.0.0 or higher (bundled with Node) | Included with Node.js |
| Git | Any recent version | https://git-scm.com |
| A modern web browser | Chrome, Edge, Firefox, or Safari | |

Verify Node.js and npm are installed by running `node --version` and `npm --version` in a terminal.

## 3. Initial Setup

### Step 1 — Clone the repository

```bash
git clone <your-gitlab-repository-url>
cd VacationProject
```

### Step 2 — Configure environment variables

Create or verify the file `server/.env` contains the following keys. **Do not commit this file to version control.**

```env
# GeoDB Cities API
RAPIDAPI_KEY=your_rapidapi_key_here
RAPIDAPI_HOST=wft-geo-db.p.rapidapi.com

# Booking.com (hotels)
BOOKING_API_KEY=your_rapidapi_key_here
BOOKING_API_HOST=booking-com.p.rapidapi.com

# Excursions (booking-com15)
EXCURSION_API_KEY=your_rapidapi_key_here
EXCURSION_API_HOST=booking-com15.p.rapidapi.com

# Flights
FLIGHT_API_KEY=your_rapidapi_key_here
FLIGHT_API_HOST=flight-fare-search.p.rapidapi.com

DEFAULT_CURRENCY=USD
```

All four APIs use the same RapidAPI key. Subscribe to each on rapidapi.com before running the app:

- `wft-geo-db.p.rapidapi.com` — GeoDB Cities (city autocomplete)
- `booking-com.p.rapidapi.com` — Booking.com (hotel search)
- `booking-com15.p.rapidapi.com` — Booking.com v15 (excursions/attractions)
- `flight-fare-search.p.rapidapi.com` — Flight search

### Step 3 — Initialize the database

Run the database initialization script once to create all tables:

```bash
cd server
node src/scripts/initDb.js
```

Expected output: `All database tables initialized!`

If the database already exists from a previous run, this command is safe to run again — it uses `CREATE TABLE IF NOT EXISTS`.

### Step 4 — Install all dependencies

From the root `VacationProject` folder, run:

```bash
cd ..
npm run install:all
```

This installs dependencies for the root package, the client, and the server in a single command.

## 4. Running the Application

### Start both client and server together

From the root `VacationProject` folder:

```bash
npm start
```

This uses `concurrently` to launch both processes simultaneously. Output is color-coded: yellow for `[SERVER]` and cyan for `[CLIENT]`.

The React app will open automatically at `http://localhost:3000`. The Express API runs on `http://localhost:5000`.

### Start client and server separately (advanced)

```bash
# Terminal 1 — Backend
cd server && node src/server.js

# Terminal 2 — Frontend
cd client && npm start
```

## 6. Database Management

### Reset the database

To wipe all data and start fresh, delete the database file and re-initialize:

```bash
rm server/database/vacation.db
node server/src/scripts/initDb.js
```

### Add the description column (if upgrading from an earlier version)

If the excursions table was created before the `description` column was added, run this migration once:

```bash
node -e "const db = require('./server/src/config/database'); db.prepare('ALTER TABLE excursions ADD COLUMN description TEXT').run(); console.log('Done')"
```

## 7. Troubleshooting

| Problem | Solution |
|---|---|
| 'Invalid API key' in server terminal | Check that `server/.env` exists and dotenv path is correct. The path in `server.js` must resolve to `server/.env` using `path.resolve(__dirname, '../.env')`. |
| 'Too many requests' from cities API | GeoDB free tier allows 1 req/sec. The rate limiter in `cities.routes.js` handles this automatically. If it persists, wait 60 seconds. |
| tripId shows as null or undefined | Ensure you are passing `tripId` as a prop (integer) from the parent component, not relying on `useParams()` in child pages. |
| 'NOT NULL constraint failed: excursions.tripId' | The tripId prop is undefined. Verify the parent component passes `tripId={parseInt(tripId, 10)}` to the child page. |
| Excursions search returns no results | Ensure you are subscribed to `booking-com15.p.rapidapi.com` on RapidAPI. Update `EXCURSION_API_HOST` in `server/.env`. |
| Navbar shows wrong trip links | Navbar reads tripId from `location.pathname`. Navigate to a trip page first; the correct ID will be stored in localStorage. |
| Port 5000 already in use | Kill the existing process: `npx kill-port 5000`. On Windows: `netstat -ano \| findstr :5000`, then `taskkill /PID <pid> /F`. |

## 8. Adding New Features

The modular architecture makes adding new domains straightforward:

1. Create `server/src/routes/newfeature.routes.js` with your Express router.
2. Register it in `server/src/routes/index.js`: `router.use('/newfeature', require('./newfeature.routes'));`
3. Add the corresponding API function to `client/src/services/api.js`.
4. Create the React page in `client/src/pages/` and register its route in `App.js`.
